2. Features & Usage

BallIdentifier currently provides the following features:

  • Identify: Right-click supported spawn messages from Ballsdex, FoodDex, HistoryDex, or JoJoDex and select "Apps → Identify" to recognize the ball and get its rarity and artist.
  • Info: Use /info to view details about a specific entry, including rarity, wave, common value, collector requirement, demand tier, and collectors when available.
  • Collectors: Use /collectors to browse collector lists for supported dexes in an interactive view.
  • Notifications: Use /notifications in DMs to manage optional thank-you messages and upvote reminder DMs.
  • Notifier: Server notifier configuration is managed in the dashboard at ballidentifier.xyz/dashboard.
  • Help & Stats: Use /help to view bot status, supported bots, usage statistics, and useful links.
3. Rate Limits

Non-upvoters are limited to 1 identification per 10 minutes. Users who currently appear in the bot's upvote cache have no identification cooldown.

Upvote status is checked against Discord Bot List and refreshed periodically. Attempting to bypass rate limits using bots, scripts, or other automated methods is prohibited and may result in restrictions.

Pro Tip: Upvoting the bot removes the identification cooldown and can enable optional upvote reminders if you turn them on.
4. Accuracy & Error Reporting

BallIdentifier uses image hashing and comparison against maintained assets to identify spawn images. When confidence is low, bot marks the result as unknown and automatically sends a verification report.

Please report inaccuracies when you see them. Feedback helps keep the asset database accurate and improves future matches.
